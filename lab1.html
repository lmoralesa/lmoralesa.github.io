Laboratory No. 1: Introduction to Kali Linux

Security Evaluation Methodologies

A penetration test (Penetration Testing) is the process used to perform a high-level security evaluation or audit. A methodology defines a set of rules, practices, procedures, and methods to follow and implement during any information security audit. A penetration testing methodology defines a roadmap with useful ideas and proven practices, which must be carefully managed to properly evaluate security systems.

Types of Penetration Testing

There are different types of penetration tests; the most common and accepted are Black-Box penetration tests, White-Box penetration tests, and Grey-Box penetration tests.

  • Black Box Testing: This type of testing takes the perspective of a malicious attacker, where testing is conducted without the knowledge of the entity's IT team, but with full knowledge and permission from management. This type of test is useful for evaluating technical security controls, the IT team's response to perceived security incidents, and staff knowledge and implementation of the entity's security policies.
  • White Box Testing: This type of testing involves conducting external and internal tests with the knowledge and consent of the entity's IT staff, allowing a complete evaluation of the network and systems' security posture. This test can also provide a training opportunity, with staff observing the activities and methods used by professionals to evaluate and potentially bypass implemented security mechanisms. This provides context for security requirements implemented or maintained by IT staff, and can also help teach IT staff how to conduct tests.
  • Grey Box Testing: The testing team has limited knowledge of defenses and assets. This type of testing evaluates the testing team's knowledge, with efficiency being the nature of the test. The depth and breadth depend on the quality and quantity of information provided to the testing team. It simulates an attack carried out by a disgruntled or dissatisfied (employee) member of the entity.

Option No. 1: Setting up a Kali Linux Virtual Machine

Activity No. 1: Download VirtualBox

  1. Update your operating system.
  2. Download Oracle Virtual Box: Go to https://www.virtualbox.org/
  3. Click on "Download VirtualBox 6.1"
  4. Select your host computer's operating system.
  5. Open the downloaded file.
  6. Run the installer and review each window and its options.
  7. Select the features you want to install and the VirtualBox location.
  8. Select the customization options you want to install.
  9. Review the network and configuration warning and proceed with the installation.
  10. Click on Install to proceed with the installation.
  11. Once the installation process is complete, you will see the VirtualBox main screen.

Activity No. 1: Download Kali Linux

  1. Go to https://www.kali.org/get-kali/
  2. Select the Virtual Machines option.
  3. Select the VirtualBox (64b) option and wait for the file to download.
  4. Locate the downloaded file and right-click to select "Open with."
  5. If you don't have a program to open .7z files, select "Look for an app in the Microsoft Store."
  6. Otherwise, use 7-Zip, WinRAR, or a similar program to extract the files.
  7. Select a location to extract the files (e.g., `D:\VirtualMachines\kali-linux-2022.3-virtualbox-amd64`).
  8. In VirtualBox, click "Add," locate the extracted files, and select the `.vbox` file.
  9. Review and adjust Virtual Machine settings (memory, processors).
  10. Click "Start" to run the virtual machine.

First Activities in Kali Linux: Updating Kali Linux

After installation, regularly check for updates. Use the following commands:

sudo apt update
